{"posts":[{"title":"使用Gridea和Github搭建自己的博客","content":" 这篇博客记录了我创建自己博客时所使用的主要工具，主要是使用Gridea和Github。不过Github的访问速度实在感人，未来我的博客可能会发表在Gitee上，令人遗憾的截止到我写这篇博客为止，Gridea暂时不支持将文章推送到Gitee上。当然Gridea的作者已经将支持Gitee提上日程了。 安装Gridea 首先在gridea的官网下载安装包，国内可以选择gitee的镜像源 进行傻瓜式安装，安装完成后打开软件，界面展示如图所示 此时我们可以通过点击右上角加号的新建文章，文章的格式是Markdown格式文件，可以通过Typora软件进行编辑，然后复制到Gridea中，值得注意的是在Typora中插入图片时，应该直接引用file![](file:path) ,而非img<img src="file:path"/>。 此时如果我们点击左下角的预览功能，我们可以在本地查看，文章转换为html后的演示，具体实验原理是使用了Hexo框架实现，如果你学习过hexo框架，那么你将清楚地明白，文章是如何被转换为html的这正是我们图片应当使用file的原因。 将文章推送到远端 当文章写好之后，文章还是存在于当前电脑，我们无法从互联网上查看我们的文章，我们可以将我们的文章部署到Github上去。Github提供Pages服务，可以帮助我们发布我们的博客。 首先我们需要一个创建一个仓库，仓库的名称是固定的，格式是Username.github.io，例如：我自己Github账户名是GuanWX，那么我的的博客仓库就是GuanWX.github.io。 之后我们需要打开这个仓库的Pages服务(事实上如果我们创建的仓库名称符合第1条中的规则时，Pages服务会自动打开)。点击仓库的setting，在下方找到Github Pages，打开服务即可。此处有一个提示，它会告诉你的博客地址是什么。我的博客地址就是guanwx.github.io 在开启Pages服务后，我们需要新建一个token。点击个人头像，在下拉菜单中点击依次Setting --> Developer Setting --> Personal access tokens，新建一个token，勾选repo权限。创建完成后不要着急关闭界面，这个token之后不在展示，只会展示这一次，一定要保存好。 点击Gridea客户端中的的远程界面，在远程见面中配置关键的信息，点击链接测试，如果配置信息没有问题的话，那么此时就会提示链接成功，点击保存即可退出。 点击Gridea左下角的同步，等待同步完成。打开我们的博客地址就会发现我们在博客已经被推送到了Github上，现在所有人都可以访问我们的博客了。 ","link":"https://guanwx.github.io/post/gridea-github/"}]}
